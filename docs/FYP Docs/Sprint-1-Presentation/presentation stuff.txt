Faculty Feedback Resolution
Feedback from Proposal Defense
“Why is there a need for such a platform when major tech giants haven’t built one?”
Resolution Implemented
Vendor incentives ≠ neutrality
Interoperability gap (OpenQASM 3.0 IR missing)
Education & research need (cross‑framework learning)
Scoped to simulation (no hardware/pulse/timing)
Feasibility proven (Iteration I + converters + tests)
-----------------------------
Feedback from Proposal Defense
"What if the other team can't build the execution platform — how will you test your code?"
Resolution Implemented
Standard IR (OpenQASM 3.0) → run on existing simulators (Aer/Cirq/PL, cloud)
Execution‑independent compiler
Validate vs native simulators (Aer/Cirq/PennyLane)
Round‑trip equivalence (source ↔ QASM ↔ source)
Static QASM validation (grammar/semantics)
Still valuable: conversion, standard QASM export, IDE/education

-----------------------------
Feedback from Proposal Defense
"Scope clarity — how much depth of each language will be covered by QCanvas?"
Resolution Implemented
Complete scope documented included in Mid‑I Report
Included: standard gates (X, Y, Z, H, S, T, RX/RY/RZ, U, CX/CZ, SWAP, gphase), modifiers (ctrl@, inv@), measurement, reset, barriers; types (qubit, bit, int, float, bool), arrays/slices, if/for, arithmetic/comparison, I/O directives
Excluded: pulse/OpenPulse, timing (delay/duration/box), extern/hardware‑specific, memory mgmt, QEC, physical qubits ($i)

Feedback from Proposal Defense
"Domain-specific challenges not mentioned"
Resolution Implemented
Qiskit: 
U/CX‑basis → map to RX/RY/RZ
classical regs vs QASM bits
Cirq: 
moment scheduling → linearize to instruction order
device/qubit ordering differences
no classical regs
PennyLane: 
high‑level ops → decompose to core gates
measurement post‑processing differs

Feedback from Proposal Defense
"How to verify conversion accuracy and translator correctness?"
Resolution Implemented
 Correctness: 
L2<1e‑8
TVD<1e‑3
fidelity>0.999999
Conformance: 100% Iteration I (reject excluded)
Performance: p95<2s (≤500 ops)
Round‑trip: source ↔ QASM ↔ source ≥ 99%
Diagnostics: precise, actionable
Coverage: ≥80%; property‑based fuzz
