\section{Conclusion}

This final report has presented the complete development of QCanvas and QSim across four iterative sprints, from initial prototype to production-ready, community-accessible platform. Every major objective set out in the original FYP specification has been achieved.

\textbf{Iteration 1} established the foundational architecture: a FastAPI backend, a Next.js frontend with Monaco editor, and the core Qiskit/Cirq/PennyLane to OpenQASM 3.0 conversion pipeline. The key insight demonstrated here was that a single, standards-based intermediate representation could serve as a universal translator for disparate quantum frameworks.

\textbf{Iteration 2} deepened the platform with advanced language features (complex control flow, subroutines, bitwise operations), expanded PennyLane gate coverage, a secure database layer (PostgreSQL, Redis, Alembic), and the hybrid CPU--QPU execution model. Security properties (AES-256 confidentiality, Bcrypt integrity, RBAC availability) were formally implemented.

\textbf{Iteration 3} transformed QCanvas from a functional tool into an intelligent, community-driven learning ecosystem through real-time D3.js circuit visualisation, the ``Quantum Explain It'' hover system, a gamification engine with 44 unique achievements, and the recipe-sharing community platform.

\textbf{Iteration 4} completed the project. Both the QCanvas converter SDK (\texttt{pip install qcanvas-sdk}) and the QSim simulation SDK (\texttt{pip install qsim-sdk}) were packaged and deployed to PyPI---work produced in this FYP is now accessible to the global quantum computing community. Previous Iterations 1--3 had proposed hardware integration, noise-aware simulation, and an expanded community platform as future directions; Iteration 4 instead prioritised SDK deployment and a rigorous benchmarking study, which were assessed as higher-impact deliverables for a final submission.

The benchmarking study confirmed that Qiskit and Cirq are statistically equivalent (QPack $\approx 58.8$ each; JSD $\approx 0$) and produce constant-depth circuits regardless of qubit count, while PennyLane exhibits super-linear depth growth for oracle algorithms (QPack $= 9.28$; scaling exponent $a = 2.21$ for Grover's). The Quantum Volume analysis confirmed all three frameworks achieve QV = 2048 on ideal simulation, demonstrating that the divergence is entirely a compiler-level artefact, not a semantic one.

In summary, QCanvas successfully delivers:
\begin{itemize}
    \item A unified quantum IDE supporting Qiskit, Cirq, and PennyLane via a single OpenQASM 3.0 pipeline.
    \item A production-ready QSim quantum simulator SDK published on PyPI with statevector, density matrix, and stabilizer backends.
    \item An educational platform with hover explanations, 44-achievement gamification, and community circuit sharing.
    \item A rigorous 12-algorithm, 3-framework, 136-configuration benchmarking study with full compilation time, structural, statistical equivalence (JSD), Quantum Volume, and QPack results.
    \item A complete software system with 304 passing tests, Docker Compose deployment, and AWS-compatible infrastructure.
\end{itemize}

\section{Future Work}

\subsection{Near-Term Engineering Goals}

\subsubsection{Hardware Backend Integration}
The pluggable QPU endpoint design allows real hardware execution without structural changes:
\begin{itemize}
    \item \textbf{IBM Quantum:} Integrate Qiskit's \texttt{IBMProvider} to dispatch compiled QASM to real QPUs.
    \item \textbf{AWS Braket:} Use Braket's OpenQASM support to target IonQ, Rigetti, and OQC backends.
\end{itemize}

\subsubsection{Noise-Aware Simulation}
\begin{itemize}
    \item Extend QSim with depolarising, amplitude damping, and phase damping noise models.
    \item Implement zero-noise extrapolation and measurement error mitigation in the execution pipeline.
\end{itemize}

\subsubsection{Expanded Framework Support}
\begin{itemize}
    \item \textbf{Q\# (Microsoft):} OpenQASM 3.0 export via Q\#'s Python interop layer.
    \item \textbf{PyQuil (Rigetti):} Quil-based circuit support.
    \item \textbf{TKET (Pytket):} Leverage pytket's built-in OpenQASM export \cite{sivarajah2020tket}.
\end{itemize}

\subsubsection{Production Infrastructure}
\begin{itemize}
    \item Migrate from Docker Compose to a Kubernetes cluster with horizontal pod autoscaling.
    \item Replace REST-based QCanvas--QSim communication with gRPC for lower latency.
    \item Add end-to-end distributed tracing via OpenTelemetry.
    \item TOTP-based multi-factor authentication.
\end{itemize}

\subsection{Long-Term Research Directions}

\subsubsection{Advanced Visualisation and Analytics}
\begin{itemize}
    \item Animated Bloch sphere showing qubit state evolution gate by gate.
    \item Entanglement graph showing qubit correlation relationships.
    \item AI-powered circuit optimisation suggestions via LLM gate analysis.
\end{itemize}

\subsubsection{Community and Research Collaboration}
\begin{itemize}
    \item Git-style circuit versioning with forking and community review.
    \item Research reproducibility tools: circuit versioning with full execution environment snapshots and LaTeX export.
    \item Machine learning-based adaptive learning system that personalises content difficulty based on user history.
    \item Tiered rate-limited public REST API with OAuth 2.0 for CI/CD quantum testing pipelines.
\end{itemize}
