# FastQSim SDK Integration Guide

This guide documents the transition from our local, in-process quantum simulator (`qsim`) to the cloud-orchestrated **FastQSim SDK** (`fastqsim`). This integration links QCanvas with the FastQubit orchestration platform, enabling cloud-based quantum circuit simulation across multiple backends (Cirq, Qiskit, and PennyLane).

---

## 🚀 Architectural Shift

Previously, QCanvas executed quantum simulations locally within the application server process using the local `qsim` library. The new integration delegates execution to the **FastQubit Orchestration Platform** via the **FastQSim client SDK**, which provides the following advantages:

- 🌐 **Offloaded Execution:** Heavy simulations run on high-performance cloud pods rather than local server threads.
- ⏱️ **Server-Side Long-Polling:** Reduces polling overhead with a highly optimized wait interface.
- 📊 **Rich Observability:** Captures hardware-level CPU/memory utilization and billing/telemetry directly from execution pods.
- 🔒 **Zero-Trust Token Auth:** Pod-scoped execution context utilizing strict secure token verification (`fqp_...`).

## ⚙️ Step 2: Environment Configuration

The FastQSim SDK reads environment variables to determine endpoint, credentials, and execution mode.

Configure the following variables in your local environment or add them to your [.env](file:///c:/Study%20Material/FYP/QCanvas-Project/QCanvas/.env) file:

```bash
# Core Authentication Token (Required - Format: fqp_...)
export FASTQUBIT_SESSION_TOKEN="fqp_your_pod_token_here"

# FastQubit API Endpoint (Optional - Defaults to production API)
export FASTQUBIT_ENDPOINT="https://api.qsim.io"

# Execution Mode (Optional - 'cloud' or 'local'. Defaults to 'cloud')
export FASTQSIM_EXECUTION_MODE="cloud"

# Client Polling Interval (Optional - Defaults to 1.0 seconds)
export FASTQSIM_POLL_INTERVAL_SECONDS="1.0"
```

> [!NOTE]
> The `FASTQUBIT_SESSION_TOKEN` is a pod-scoped token specifically generated by the FastQubit orchestration system. Ensure it is kept secret and not committed to source control.

```python
class SimulationService:
    """Service for executing quantum simulations using FastQSim platform"""
    
    def __init__(self):
        self.available_backends = ['cirq', 'qiskit', 'pennylane']
        self.legacy_backends = ["statevector", "density_matrix"]
        self.client = None
        
        if FASTQSIM_AVAILABLE:
            try:
                # Initialize the FastQSim global client
                self.client = fastqsim.init()
                print("✓ FastQSim client initialized successfully")
            except Exception as e:
                print(f"⚠️ FastQSim client initialization failed: {e}")
```

#### 3. Execution Wrapper Method (`execute_qasm_with_qsim`)

```python
    def execute_qasm_with_qsim(self, qasm_code: str, backend: str = "cirq", shots: int = 1024) -> Dict[str, Any]:
        """
        Execute OpenQASM code using FastQSim SDK Client
        
        Args:
            qasm_code: OpenQASM 3.0 code to execute
            backend: Simulation backend (cirq, qiskit, or pennylane)
            shots: Number of measurement shots
            
        Returns:
            Dictionary containing simulation results compatible with QCanvas frontend
        """
        try:
            if not FASTQSIM_AVAILABLE:
                return {
                    "success": False,
                    "error": "FastQSim SDK is not installed."
                }
                
            # Attempt lazy initialization if client is not ready
            if self.client is None:
                try:
                    self.client = fastqsim.init()
                except Exception as e:
                    return {
                        "success": False,
                        "error": f"Failed to initialize FastQSim client: {str(e)}. Please check FASTQUBIT_SESSION_TOKEN."
                    }

            # Validate backend compatibility
            if backend not in self.available_backends:
                return {
                    "success": False,
                    "error": f"Unsupported backend: {backend}. Choose from: {self.available_backends}"
                }

            # Sanitize user QASM code
            normalized_qasm = self._sanitize_qasm_input(qasm_code)
            if not normalized_qasm:
                return {"success": False, "error": "QASM input is empty."}
            if "OPENQASM" not in normalized_qasm.upper():
                return {"success": False, "error": "Invalid OpenQASM input: missing OPENQASM header."}

            start_time = time.perf_counter()
            
            # --- FastQSim Client Execution ---
            # Submits the OpenQASM 3.0 circuit to FastQubit and waits synchronously
            job = self.client.run(
                normalized_qasm,
                backend=backend,
                shots=shots,
                asynchronous=False  # Blocks client thread until job completes
            )
            
            # Retrieve completed job results
            job_result = job.result()
            execution_time_ms = (time.perf_counter() - start_time) * 1000

            # --- Result Format Mapping ---
            # Construct a response perfectly compatible with the existing frontend
            result_dict = {
                "counts": job_result.counts,
                "probs": job_result.probabilities,
                "metadata": {
                    "n_qubits": job.num_qubits or self._extract_qubit_count(normalized_qasm),
                    "backend": backend,
                    "visitor": f"fastqsim_{backend}_visitor",
                    "shots": shots,
                    "job_id": job.job_id,
                    "status": job.status.value if hasattr(job.status, 'value') else str(job.status),
                    "execution_time": f"{execution_time_ms:.2f}ms",
                    "simulation_time": f"{job.execution_time_seconds * 1000 if job.execution_time_seconds else execution_time_ms:.2f}ms",
                    "memory_usage": f"{job.peak_memory_mb:.2f}MB" if job.peak_memory_mb else "N/A",
                    "cpu_usage": f"{job.cpu_seconds_total:.2f}s" if job.cpu_seconds_total else "N/A",
                    "fidelity": 100.0
                }
            }
            
            # Convert any leftover NumPy types securely
            result_dict = convert_numpy_types(result_dict)

            return {
                "success": True,
                "results": result_dict,
                "backend": backend,
                "shots": shots
            }

        except Exception as e:
            import traceback
            print(f"FastQSim Execution Failure: {traceback.format_exc()}")
            return {
                "success": False,
                "error": f"FastQSim execution failed: {str(e)}"
            }
```

## 🛠️ Troubleshooting & Gotchas

1. **Authentication Failures (HTTP 401 / AuthenticationError):**
   * Ensure `FASTQUBIT_SESSION_TOKEN` begins with the `fqp_` prefix.
   * Verify token expiry or pod lifespan limits.

2. **Validation Errors on Circuit Code:**
   * FastQSim SDK only accepts valid **OpenQASM 3.0** string code.
   * Make sure native Python library structures (`cirq.Circuit`, `QuantumCircuit`) are compiled to OpenQASM *before* submission.

3. **Job Timeout Exceptions:**
   * Large circuit profiles may exceed synchronous run thresholds.
   * In the future, asynchronous polling using `client.wait(job.job_id)` can be implemented with a worker/task-queue system.
