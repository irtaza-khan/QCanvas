FastQSim Python SDK

A thin, zero-dependency HTTP client for submitting OpenQASM 3.0 circuits to the FastQSim quantum-simulation platform and retrieving results. The SDK handles job lifecycle management, server-side long-polling, parallel batch submission, and automatic session provisioning — you only need to provide a circuit and call run().

Design boundary The SDK accepts only OpenQASM 3.0 strings. Native circuit objects from Qiskit, Cirq, or PennyLane are not accepted. Convert your circuit to QASM 3.0 before calling any SDK method.
Session management is automatic Sessions are provisioned transparently by the FastQubit platform when you authenticate. You never need to create or manage sessions yourself. Simply call fastqsim.init() and start submitting circuits.

Installation

pip install fastqsim

Or install from source in editable mode (for development):

pip install -e ./fastqsim

Python: 3.8 or later. Dependencies: requests >= 2.28 only.

Runtime Context

In QCanvas sessions, all required variables are injected automatically. For standalone use, set them in your environment before calling fastqsim.init().

Variable Required Description
FASTQUBIT_USER_ID Yes Caller identity forwarded as X-User-Id on every request. Injected automatically in QCanvas pods.
FASTQSIM_API_TOKEN No Bearer token for authenticated endpoints. If present, extracted sub/user_id/uid claim is used as the user ID.
FASTQUBIT_ENDPOINT No API base URL. Defaults to https://api.qsim.io.
FASTQSIM_EXECUTION_MODE No cloud (default) or local. Local mode disables polling and cloud operations.
FASTQUBIT_SESSION_ID / FASTQSIM_SESSION_ID No Pin a specific session ID instead of resolving one automatically.
FASTQSIM_POLL_INTERVAL_SECONDS No Fallback sleep between long-poll retries. Defaults to 1.0.

Quick Start

import fastqsim

client = fastqsim.init()   # reads FASTQUBIT_USER_ID from environment

qasm = """
OPENQASM 3.0;
include "stdgates.inc";
qubit[2] q;
bit[2]   c;

h  q[0];
cx q[0], q[1];
c = measure q;
"""

# Synchronous — blocks until the job finishes
job    = client.run(qasm, shots=2048, asynchronous=False)
result = job.result()

print(job.job_id, job.status.value)   # e.g. "3fa8... completed"
print(result.counts)                  # {"00": 1024, "11": 1024}

What's New

Recent improvements
  • Server-side long-polling — wait() and synchronous run() now pass ?wait=30 to the API, so the server holds the connection via Redis pub/sub and returns the moment a job completes. Network round-trips drop from ~60/min to ~2/min for typical jobs.
  • True parallel batch — run_batch() now submits circuits concurrently using a thread pool. The max_parallel parameter controls actual concurrency.
  • Session ID caching — The session ID is resolved once per client instance and reused for all subsequent job submissions. No extra API calls per job.
  • Reduced round-trips after polling — After a terminal status is detected, the last fetched job record is reused directly. No redundant second fetch.

Usage Scenarios

Synchronous Run

Block until the job completes and inspect results inline. Best for interactive notebooks and short circuits.

job = client.run(
    qasm,
    backend="qiskit",
    shots=1024,
    asynchronous=False,   # default is False — blocks here
)

result = job.result()
print(result.counts)      # {"00": 512, "11": 512}
print(job.execution_time_seconds)  # e.g. 0.34

Async / Fire-and-Forget

Submit a job and do other work while it runs. Retrieve results later with wait() or get().

job = client.run(qasm, shots=4096, asynchronous=True)
print(f"Queued: {job.job_id}")

# ... do other work ...

# Block until terminal, returns updated Job
completed_job = client.wait(job.job_id, timeout=300)
result = completed_job.result()
print(result.counts)

Batch Execution

Submit multiple circuits in parallel. All jobs share the same backend, device, and shots. Use client.wait() to block on all of them.

circuits = [qasm_bell, qasm_ghz, qasm_qft]

# Submitted concurrently (up to max_parallel threads)
jobs = client.run_batch(
    circuits,
    shots=1024,
    max_parallel=3,   # controls client-side thread concurrency
)

print([j.job_id for j in jobs])

# Wait for all to complete
completed = client.wait([j.job_id for j in jobs], timeout=600)

for job in completed:
    print(job.job_id, job.result().counts)

Cancel a Job

Cancellation is idempotent — cancelling an already-terminal job is a no-op and returns the current state.

job = client.run(qasm, asynchronous=True)

# Request cancellation
updated_job = client.cancel(job.job_id)
print(updated_job.status.value)   # "cancelled" or already-terminal status

# Cancel via the Job object directly
job2 = client.run(qasm, asynchronous=True)
job2.cancel()
print(job2.status.value)
Note Cancellation signals the platform to stop scheduling this job but does not kill an already-running worker pod. If a worker is mid-execution it may finish, but any terminal state updates will be ignored once cancelled.

Deadline-Safe Polling

Use wait() with a timeout and catch JobTimeoutError for robust production workflows.

from fastqsim import JobTimeoutError, JobFailedError

job = client.run(qasm, asynchronous=True)

try:
    completed = client.wait(job.job_id, timeout=120)
    result = completed.result()
    print(result.counts)
except JobTimeoutError:
    print(f"Job {job.job_id} did not finish within 120 s")
    # Optionally cancel it
    client.cancel(job.job_id)
except JobFailedError as exc:
    print(f"Job failed: {exc}")

Search Job History

Query your job history with optional filters. Returns up to limit jobs ordered most-recently-updated first.

from datetime import datetime, timezone, timedelta

# All completed jobs in the last 24 hours
yesterday = datetime.now(timezone.utc) - timedelta(days=1)

jobs = client.search(
    run_status="completed",
    start_date=yesterday,
    limit=20,
)

for job in jobs:
    print(job.job_id, job.status.value, job.execution_time_seconds)

# Filter by backend
qiskit_jobs = client.search(backend="qiskit", limit=50)

# Failed jobs for investigation
failed = client.search(run_status="failed", limit=10)
for job in failed:
    print(job.job_id, job.error_message)

Statevector Retrieval

Request a full statevector simulation. The statevector is returned as a list of complex values.

job = client.run(
    qasm,
    shots=0,               # No measurement shots needed
    simulation_type="statevector",
    asynchronous=False,
)

sv = job.get_statevector()
print(f"State dimension: {len(sv)}")   # 2^n_qubits
print(f"|00⟩ amplitude: {sv[0]:.4f}")  # e.g. (0.7071+0j)
print(f"|11⟩ amplitude: {sv[3]:.4f}")  # e.g. (0.7071+0j)
Large circuits For large qubit counts the statevector may exceed the inline-response limit. In that case it is stored as an external artifact and get_statevector() fetches it from S3 automatically.

External Artifact Download

When job.has_external_artifact is True, the full result JSON was uploaded to S3. job.result() fetches and merges it automatically.

job = client.run(large_circuit_qasm, shots=8192, asynchronous=False)

if job.has_external_artifact:
    # result() transparently downloads the S3 artifact
    result = job.result()
    print(result.counts)
    # Statevector, density matrix, or probabilities may also be in artifact
    if result.statevector:
        print(f"Statevector length: {len(result.statevector)}")
else:
    # Results fit inline — no extra download needed
    print(job.result_counts)

Density Matrix & MPS Simulation

Density matrix — open-system / noisy simulation. Supports all gates.

job = client.run(
    qasm,
    simulation_type="density_matrix",
    shots=1024,
    asynchronous=False,
)
result = job.result()
print(result.counts)
# Density matrix available in result.density_matrix (if returned by backend)

MPS (Matrix Product State) — efficient for low-entanglement circuits with many qubits.

job = client.run(
    large_qasm,
    simulation_type="mps",
    shots=2048,
    backend="qiskit",
    asynchronous=False,
)
print(job.result().counts)

CHP (Stabilizer / Clifford) — exponentially faster for Clifford-only circuits.

job = client.run(
    clifford_qasm,
    simulation_type="chp",
    shots=4096,
    asynchronous=False,
)
print(job.result().counts)

Error Handling

All SDK exceptions inherit from FastQSimError. Import only what you need.

from fastqsim import (
    FastQSimConnectionError,
    AuthenticationError,
    QuotaExceededError,
    ValidationError,
    JobFailedError,
    JobTimeoutError,
    JobNotFoundError,
    JobCouldNotCancelError,
)

try:
    client = fastqsim.init()
except FastQSimConnectionError as exc:
    print(f"Cannot reach API: {exc}")
    raise SystemExit(1)
except AuthenticationError as exc:
    print(f"Auth failed — check FASTQUBIT_USER_ID: {exc}")
    raise SystemExit(1)

try:
    job = client.run(
        invalid_qasm,          # Bad QASM
        shots=-5,              # Invalid shots
        asynchronous=False,
    )
except ValidationError as exc:
    print(f"Bad request: {exc}")

try:
    job = client.run(good_qasm, shots=1024, asynchronous=False)
    result = job.result()
except QuotaExceededError:
    print("Monthly quota exhausted — upgrade your plan")
except JobFailedError as exc:
    print(f"Simulation failed: {exc.reason}")
except JobTimeoutError as exc:
    print(f"Job {exc.job_id} timed out after {exc.timeout}s")
except JobNotFoundError as exc:
    print(f"Job not found: {exc}")

Job Metadata & Timing

Every Job object carries full lifecycle timestamps and resource-usage metrics. 

job = client.run(qasm, shots=1024, asynchronous=False,
                 job_name="bell-state-test",
                 tags={"experiment": "v3", "team": "qcomp"},
                 metadata={"circuit_version": "2025-03-31"})

# Lifecycle timestamps (all datetime, UTC)
print(job.queue_enqueued_at)      # when job was published to queue
print(job.execution_started_at)   # when worker pod began simulation
print(job.execution_finished_at)  # when worker pod finished
print(job.completed_at)           # when terminal state was committed

# Performance metrics
print(f"Wall time:   {job.execution_time_seconds:.3f}s")
print(f"CPU time:    {job.cpu_seconds_total:.3f}s")
print(f"Peak RAM:    {job.peak_memory_mb:.1f} MB")

# Billing telemetry
print(f"CPU millicore-s: {job.billing_cpu_millicore_seconds:.2f}")
print(f"Memory GB-s:     {job.billing_memory_gb_seconds:.4f}")

# Tags and metadata round-trip
print(job.tags)       # {"experiment": "v3", "team": "qcomp"}
print(job.metadata)   # {"circuit_version": "2025-03-31"}

# Refresh from server (e.g. for async jobs)
job.refresh()
print(job.status.value)

Global Functions

fastqsim.init(endpoint=None, execution_mode=None, timeout=300, max_retries=3) → FastQSimClient

Create and store the global client. Call once at startup. All identity and endpoint settings are sourced from environment variables — no arguments needed in QCanvas sessions.

Raises AuthenticationError if no user identity is available, and FastQSimConnectionError if the API endpoint is unreachable.

client = fastqsim.init()

# Override for testing against a local server
client = fastqsim.init(endpoint="http://localhost:8080")
fastqsim.reset() → None

Clear the global client so init() can be called again without a warning. 

fastqsim.reset()
fastqsim.init(endpoint="https://staging.api.qsim.io")

FastQSimClient

The primary class. Instantiate via fastqsim.init() or directly if you need multiple clients.

from fastqsim import FastQSimClient
client = FastQSimClient(endpoint="https://api.qsim.io", timeout=120)

run()

client.run(circuit, *, backend="qiskit", device="cpu", shots=1024, simulation_type=None, seed=None, metadata=None, asynchronous=False, job_name=None, options=None, tags=None) → Job | List[Job]

Submit one circuit (or a list of circuits, delegating to run_batch) for simulation.

Parameter Type Default Description
circuit str | List[str] — OpenQASM 3.0 string. A list delegates to run_batch(). Native circuit objects are not accepted.
backend str "qiskit" Simulation backend. One of "qiskit", "cirq", "pennylane".
device str "cpu" Compute device. One of "cpu", "gpu", "quantum".
shots int 1024 Number of measurement repetitions. Must be > 0.
simulation_type str | None None One of "statevector", "density_matrix", "mps", "chp". None uses the backend default (statevector).
seed int | None None RNG seed for reproducible results. When set, identical circuits produce identical counts.
metadata dict {} Arbitrary key-value metadata stored with the job. Round-trips in job.metadata.
asynchronous bool False Return immediately (True) or block until the job completes (False). When False, uses server-side long-polling.
job_name str | None None Human-readable label for the job.
options dict {} Backend/device-specific options forwarded to the simulator.
tags dict {} Key-value tags for filtering in search().

run_batch()

client.run_batch(circuits, *, backend="qiskit", device="cpu", shots=1024, max_parallel=10, simulation_type=None, seed=None, metadata=None, job_name=None, options=None, tags=None) → List[Job]

Submit multiple circuits concurrently. Jobs are always asynchronous — the method returns as soon as all circuits are submitted. Use client.wait() to block for completion.

Parameter Type Default Description
circuits List[str] — List of OpenQASM 3.0 strings. All must be strings; mixed types raise ValidationError.
max_parallel int 10 Maximum concurrent submission threads. Set lower to avoid overwhelming the API.
All other parameters are identical to run() and apply to every circuit in the batch.

get()

client.get(job_ids) → Job | List[Job]

Fetch the current metadata for one or more jobs without waiting. Accepts a single job ID string, a Job object, or a list of either. Returns a single Job for a single input, a list otherwise.

job  = client.get("3fa8-...")
jobs = client.get(["3fa8-...", "8b2c-..."])

wait()

client.wait(job_ids, *, timeout=None, poll_interval=None) → Job | List[Job]

Block until one or more jobs reach a terminal state (completed, failed, or cancelled). Uses server-side long-polling — each poll holds the connection for up to 30 s before timing out server-side.

Parameter Type Default Description
job_ids str | Job | List[...] — Job ID(s) or Job object(s) to wait for.
timeout int | None None Maximum seconds to wait. Raises JobTimeoutError when exceeded. None waits indefinitely.
poll_interval float | None 1.0 Fallback sleep between long-poll retries (used only when the server returns before 30 s and the job is not yet terminal).

cancel()

client.cancel(job_ids) → Job | List[Job]

Request cancellation of one or more jobs. Idempotent — cancelling an already-terminal job returns it unchanged. Raises JobCouldNotCancelError if the server rejects the request.

job = client.cancel("3fa8-...")
print(job.status.value)  # "cancelled" or existing terminal status

search()

client.search(run_status=None, created_later_than=None, limit=50, backend=None, start_date=None, end_date=None) → List[Job]

Query job history for the authenticated user. Results are ordered most-recently-updated first.

Parameter Type Description
run_status str | None Filter by lifecycle status string, e.g. "completed", "failed", "running".
backend str | None Filter by backend name ("qiskit", "cirq", "pennylane").
start_date / end_date datetime | None Filter jobs by creation time range (inclusive).
created_later_than str | datetime | None Alias for a lower-bound creation-time filter.
limit int Maximum results to return. Range: 1–500. Default: 50.

Methods

job.result(timeout=None) → Result

Block until terminal, then return a Result object. Raises JobFailedError or JobTimeoutError on failure. If has_external_artifact is set, downloads from S3 automatically.

job.wait_for_completion(timeout=None, poll_interval=None) → Job

Poll until terminal and return self (updated in place). Does not raise on failure status — check job.status afterward.

job.get_counts() → Dict[str, int]

Return measurement outcome counts. Raises JobNotCompleteError if not yet terminal, JobFailedError if failed, JobDoesNotContainCountsError if no counts are present.

job.get_statevector() → List[complex]

Return the final statevector. Raises JobStatevectorTooLargeError if not available in the response or artifact. Requires simulation_type="statevector".

job.cancel() → bool

Cancel this job. Returns True if successfully cancelled.

job.refresh() → None

Re-fetch job metadata from the API and update this object in place.

Result API

A Result is returned by job.result(). All fields are optional and depend on the simulation type and backend.

Property Type Description
counts Dict[str, int] Measurement outcome counts, e.g. {"00": 512, "11": 512}.
probabilities Dict[str, float] | None Outcome probabilities. Available when returned by the backend.
statevector List[complex] | None Final state amplitude vector (statevector simulation only).
density_matrix Any | None Density matrix (density_matrix simulation only).
metadata dict Merged backend-emitted metadata from inline response and S3 artifact.
execution_time float | None Wall-clock duration in seconds (derived from timestamps).
result.get_counts(key=None) → Dict[str, int]

Return result.counts, or a register-specific counts sub-dict when key is provided.

RunArgs Helper

RunArgs lets you build a reusable argument bundle and unpack it into run().

from fastqsim import RunArgs

args = RunArgs(backend="qiskit", shots=2048, simulation_type="statevector")

# Unpack into run()
job = client.run(qasm, **args.to_dict())

JobStatus

Value Description
created Record created; pre-queue.
queued Published to RabbitMQ; awaiting worker pickup.
running Worker pod has started execution.
completed Simulation finished; results available. Terminal.
failed Simulation encountered an error. See job.error_message. Terminal.

SimulationType

Value Best for

Exceptions

All SDK exceptions derive from FastQSimError.

FastQSimError
├── FastQSimConnectionError    # Cannot reach the API endpoint
├── AuthenticationError        # Missing or invalid user identity / token
├── UnauthorizedAccessError    # Caller lacks permission for the resource
├── ValidationError            # Bad request parameters (circuit, shots, sim_type…)
├── QuotaExceededError         # Monthly simulation quota exhausted
├── APIError                   # Unexpected API response
├── InvalidDeviceTypeError     # Unknown device string
├── JobNotFoundError           # Job ID does not exist
├── JobFailedError             # Job reached FAILED status
│     .job_id  .reason
├── JobTimeoutError            # Polling deadline exceeded
│     .job_id  .timeout
├── JobNotCompleteError        # Result requested before terminal state
│     .job_id  .current_status
├── JobCouldNotCancelError     # Server rejected cancellation
│     .job_id  .reason
├── JobDoesNotContainCountsError  # No measurement counts on completed job
└── JobStatevectorTooLargeError   # Statevector not available in response

HTTP Error Mapping

HTTP Status Exception
401 AuthenticationError
402 QuotaExceededError
403 UnauthorizedAccessError
404 JobNotFoundError
409 (cancel) JobCouldNotCancelError
422 ValidationError
429, 500, 502, 503, 504 Retried automatically (up to max_retries), then raise_for_status()

Behavior Notes

  • Server-side long-polling: All polling calls pass ?wait=30 to the API. The server holds the HTTP connection open via Redis pub/sub and returns immediately when the job reaches a terminal state. Fallback: if the server returns before 30 s and the job is not yet terminal, the client sleeps poll_interval (default 1 s) before retrying.
  • Session transparency: Sessions are provisioned automatically by the FastQubit platform when you authenticate. The SDK resolves and caches the session ID on first use — subsequent job submissions in the same client instance reuse it without additional API calls.
  • Retry logic: HTTP 429 / 5xx responses are retried up to max_retries times (default 3) with exponential back-off (0.5 s, 1 s, 2 s, …). Only GET, POST, and DELETE methods are retried. 4xx errors (except 429) are not retried.
  • Circuit qasm: Only OpenQASM 3.0 strings are accepted. The circuit is stored server-side and is never echoed back in any API response.
  • Result storage: Small results are returned inline in result_counts. Large results are stored in S3 and job.has_external_artifact is True. job.result() fetches and merges S3 artifacts automatically.
  • Presigned URLs: The S3 result URL returned by the platform expires in 15 minutes. Calling job.result() always issues a fresh URL.
