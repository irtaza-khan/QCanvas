                    QCanvas Quantum Simulation Platform
                           Setup & Run Guide

OVERVIEW
--------
QCanvas is a quantum circuit simulation platform with React frontend and Python 
quantum execution backend. It supports real quantum circuit execution using 
Qiskit, Cirq, and other quantum frameworks.

PREREQUISITES
-------------

1. Node.js & npm
   - Download from: https://nodejs.org/
   - Required version: 18 or higher
   - npm comes automatically with Node.js

2. Python (for quantum execution)
   - Download from: https://python.org/downloads/
   - Required version: 3.8 or higher
   - Make sure Python is added to system PATH

3. Git
   - Download from: https://git-scm.com/downloads/
   - Required for version control

INSTALLATION STEPS
------------------

1. Navigate to Project Directory
   Open terminal/command prompt and navigate to the project folder:
   
   cd /path/to/your/qcanvas-project

2. Install Node.js Dependencies
   Run this command in the project root:
   
   npm install

3. Install Python Quantum Libraries
   Install required quantum computing frameworks:
   
   pip install qiskit qiskit-aer cirq
   
   Or if you use pip3:
   pip3 install qiskit qiskit-aer cirq

4. Verify Installation (Optional)
   Test the quantum executor:
   
   python server/quantum_executor.py

RUNNING THE PROJECT
-------------------

Development Mode (Recommended for testing):
   npm run dev
   
   This will:
   - Start React frontend with hot reload
   - Start Express backend API
   - Usually runs on http://localhost:8080
   - Check terminal output for exact URL

Production Mode:
   npm run build
   npm start

AVAILABLE FEATURES
------------------

Frontend Components:
- Code Editor: Write quantum circuits in Qiskit, Cirq, AWS Braket
- Examples Library: Pre-built quantum algorithms
- Results Visualization: Interactive histograms and measurements
- Responsive Design: Works on desktop and mobile

Backend Features:
- Real quantum execution using Python libraries
- API endpoint: POST /api/quantum/execute
- Automatic fallback to mock simulation if Python unavailable
- Security features: code sanitization, execution timeouts

Example Algorithms Included:
- Bell State Creation (quantum entanglement)
- Grover's Algorithm (quantum search)
- Quantum Fourier Transform
- Quantum Teleportation
- Deutsch-Jozsa Algorithm

TROUBLESHOOTING
---------------

Python Issues:
- Verify Python version: python --version
- Check quantum libraries: pip list | grep qiskit
- App automatically falls back to mock simulation if Python fails

Node.js Issues:
- Check Node version: node --version (should be 18+)
- Clear and reinstall: rm -rf node_modules && npm install

Port Conflicts:
- Dev server automatically finds available port
- Check terminal output for actual URL

Common Commands:
- npm run dev         : Start development server
- npm run build       : Build for production
- npm run start       : Start production server
- npm run test        : Run tests
- npm run typecheck   : Validate TypeScript

USAGE INSTRUCTIONS
------------------

1. Start the development server with: npm run dev

2. Open your browser to the URL shown in terminal (usually localhost:8080)

3. You'll see the QCanvas interface with:
   - Code editor on the left
   - Examples panel on the right
   - Results panel below examples

4. Try running an example:
   - Click on "Bell State Creation" in the examples panel
   - Click the "🚀 Run Circuit" button
   - See results in the Output/Histogram tabs

5. Write your own quantum code:
   - Select framework (Qiskit, Cirq, AWS Braket)
   - Write code in the editor
   - Adjust shots and backend settings
   - Click Run Circuit

ARCHITECTURE
------------

Project Structure:
- client/           : React frontend components
- server/           : Express backend and Python quantum executor
- shared/           : Shared TypeScript interfaces
- tailwind.config.ts: Custom quantum-themed styling

Key Files:
- server/quantum_executor.py  : Python quantum circuit execution
- server/routes/quantum.ts    : API endpoint for quantum execution
- client/pages/QCanvas.tsx    : Main application page
- client/components/          : UI components

API Endpoint:
POST /api/quantum/execute
{
  "code": "# Your quantum code",
  "shots": 1024,
  "backend": "qasm_simulator", 
  "framework": "qiskit"
}

DEPLOYMENT
----------

For production deployment:
1. Ensure Python and quantum libraries are installed on server
2. Run: npm run build && npm start
3. Consider using Docker for better isolation
4. Configure security policies for code execution

GETTING HELP
------------

- Check the terminal output for error messages
- Verify all prerequisites are installed correctly
- Ensure Python quantum libraries are working: python -c "import qiskit; print('OK')"
- The app will work with mock simulation even if Python fails
